# 📊 Intern Performance Analytics & Insights Dashboard

## 🔹 Project Overview
This project focuses on analyzing intern performance data and building an interactive dashboard to generate meaningful business insights.

It covers the complete data analytics workflow including data cleaning, exploratory data analysis (EDA), KPI definition, and dashboard development. Additionally, a Flask-based web application is implemented to simulate real-time data collection.

---

## 🎯 Objective
- Analyze intern performance data  
- Identify key performance indicators (KPIs)  
- Build an interactive dashboard for decision-making  
- Provide actionable insights to improve intern performance and placement rate  

---

## 🛠️ Tools & Technologies
- Python (Pandas, NumPy)  
- Power BI  
- Flask (for real-time data collection)  
- Jupyter Notebook

## 🚀 Sprint Details

### 🟢 Sprint 1 – Data Collection & Preparation
- Defined dataset structure  
- Collected and prepared sample data  
- Performed data cleaning  
- Handled missing values and corrected data types  
- Documented preprocessing steps  

---

### 🟡 Sprint 2 – Exploratory Data Analysis (EDA)
- Performed statistical analysis  
- Identified trends and patterns  
- Created visualizations  
- Generated key insights  
- Engineered features like:
  - Risk Status  
  - Productivity Score  
  - Performance Level  

---

### 🔵 Sprint 3 – KPI Definition & Dashboard Development
- Defined key performance indicators (KPIs):
  - Placement Rate  
  - At-Risk Percentage  
  - Average Productivity  
  - High Performers Count  

- Built an interactive Power BI dashboard  
- Designed multiple analytical pages:
  - Overview  
  - Department Analysis  
  - Top Performers  
  - At-Risk Interns  
  - Candidate Profiles  

- Developed a Flask-based web form to collect and update intern data in real time  

---

## 📊 Dashboard Features
- High-level performance overview  
- Department-wise comparison  
- Ranking of top-performing interns  
- Identification of at-risk interns  
- Detailed individual performance analysis  

---

## 💡 Key Insights
- A high percentage of interns fall under the at-risk category  
- Placement rate is relatively low and needs improvement  
- Performance varies across departments  
- High productivity does not always indicate high performance

## 🌐 Real-Time Data Collection (Flask)
A Flask-based web application is used to simulate real-time data collection.

- Users can input intern details through a web form  
- Data gets processed and stored dynamically  
- This enables a more practical, real-world data pipeline approach
